# my_tiny_malloc - моя реализация malloc и free
### Цель проекта
Разобраться, как работает управление памятью в runtime, и реализовать собственный учебный аллокатор с интерфейсом, аналогичным malloc/free

### Поддерживаемый функционал
- `my_tiny_malloc`, `my_tiny_free`
- Выравнивание памяти (8 байт)
- Coalescing (Слияние соседних free-блоков) за O(1)
- Защита от double free
- Статистика использования памяти
- Разные стратегии выделения (first-fit / best-fit)
- Динамические запросы и возвращения памяти ОС через mmap и munmap

### Ограничения
- Не является thread safe
- Нет realloc и calloc

### Компиляция:
* `gcc -DALLOC_STRATEGY_BEST_FIT ...` для использования стратегии best fit

По умолчанию, без этого флага используется first fit стратегия. Для указания best fit стратегии при компиляции можно также разкомментировать строку в файле `my_tiny_alloc.h`:
- `//#define ALLOC_STRATEGY_BEST_FIT`

### Счётчики
- `counter_total_allocated` - Сколько памяти в текущий момент занято
- `counter_total_free` - Сколько памяти в текущий момент свободно
- `counter_number_of_blocks` - Общее число блоков в памяти (занятый и свободных)
- `counter_peak_usage` -  Максимальный размер одного выделенного блока за всё время работы
- `counter_total_chunks` - Количество областей mmap
